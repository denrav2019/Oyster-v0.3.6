# Oyster Language Specification v0.1.2

## 1. Формат инструкций байт-кода

**Все инструкции строго параметрические:**

| Тип | Формат | Пример |
|-----|--------|--------|
| Параметрическая | `<X:YYYYYYYY>` | `<i:0000002A>` |
| Непараметрическая (заглушка) | `<X:00000000>` | `<P:00000000>` |

### Длина опкода

**Опкод (X) — всегда 1 символ ASCII** (например, `i`, `P`, `L`, `l`).

Это позволяет VM использовать **таблицу диспетчеризации** размером 256:
- `vm->handlers[opcode]` — мгновенный доступ к обработчику
- Без `switch/case` или `strcmp`

|------------|----------|----------|
| `i:XXXXXXXX` | значение | Загрузить целое число |
| `s:XXXXXXXX` | индекс в пуле | Загрузить строку из пула |
| `g:XXXXXXXX` | индекс переменной | Загрузить глобальную переменную |
| `p:XXXXXXXX` | индекс переменной | Сохранить в глобальную переменную |
| `S:00000000` | — | Создать ByteArray (длина на стеке) |
| `L:00000000` | — | Загрузить длину в байтах |
| `l:XXXXXXXX` | количество бит | Сдвиг влево на N бит |
| `r:XXXXXXXX` | количество бит | Сдвиг вправо на N бит |
| `U:XXXXXXXX` | индекс | Установить элемент массива |
| `E:XXXXXXXX` | индекс | Получить элемент массива |
| `P:00000000` | — | Печать (print) |
| `a:00000000` | — | Сложение |
| `G:00000000` | — | Сравнение `>` |
| `j:XXXXXXXX` | смещение | Безусловный переход |
| `f:XXXXXXXX` | смещение | Переход если false |
| `E:00000000` | — | Завершение программы |

| `e:00000000` | — | Равно (==) — числа и строки (лексикографически) |
| `n:00000000` | — | Не равно (!=) — числа и строки (лексикографически) |
| `l:00000000` | — | Меньше (<) — числа и строки (лексикографически) |
| `g:00000000` | — | Больше (>) — числа и строки (лексикографически) |
| `b:00000000` | — | Меньше или равно (<=) — числа и строки (лексикографически) |
| `c:00000000` | — | Больше или равно (>=) — числа и строки (лексикографически) |

**Поведение сравнений:**

- Если оба операнда — числа (`V_INTEGER`): сравнение по значению.
- Если оба операнда — строки (`V_STRING`): **лексикографическое сравнение** (как `strcmp`).
- Если типы разные: возвращается `undef` (можно позже добавить преобразование).

## 4. ByteArray — основа строк и массивов

**И строки, и массивы используют единую структуру данных `ByteArray`:**

```c
typedef struct {
    uint8_t* data;   // указатель на данные
    int length;      // длина в байтах (фиксированная!)
} ByteArray;

Важнейшее свойство:
Длина ByteArray задаётся один раз при создании и никогда не меняется.

Нельзя добавить элемент (push отсутствует)

Нельзя удалить элемент (pop отсутствует)

Нельзя изменить длину

Можно только читать и перезаписывать существующие элементы

Это делает строки и массивы предсказуемыми и эффективными с точки зрения памяти.

## 6. Переменные
Префикс	Тип	Описание
$	Скаляр	Числа, строки, undef
@	Массив	Только массивы (ByteArray)

7. Синтаксис (примеры)

# Скаляры
$x = 42
$name = "Oyster"

# Массивы (только @)
@arr = [10, $x, "World", 99]

# Длина массива — фиксирована
print(len(@arr))   # 4

# Доступ и изменение
print(@arr[0])     # 10
@arr[0] = 100
print(@arr[0])     # 100

# Нельзя добавить элемент
# push(@arr, 5)   # <-- ОШИБКА: массив фиксированной длины

## 8. Архитектура VM
Таблица диспетчеризации


typedef void (*InstrHandler)(VM* vm, uint32_t param);
InstrHandler handlers[256];

// Регистрация
handlers['i'] = handle_integer;
handlers['P'] = handle_print;
handlers['l'] = handle_shift_left;   // параметр — количество бит
handlers['r'] = handle_shift_right;  // параметр — количество бит

// Вызов
handlers[opcode](vm, param);



## 9. Принципы
Все инструкции — строго параметрические (<X:YYYYYYYY>).

Опкод — всегда 1 символ ASCII (для таблицы диспетчеризации).

Никаких непараметрических инструкций (всегда используем <X:00000000>).

Строки — через $.

Массивы — только через @.

Длина ByteArray — фиксирована на всё время жизни.

Все изменения сверяем со SPEC.md.

typedef enum {

[0] — количество элементов

5. Компилятор

5.1. Приоритеты операторов (от низшего к высшему)
Уровень	Операторы
0	or
1	and
2	| (битовое OR)
3	^^ (битовое XOR)
4	& (битовое AND)
5	+, -
6	*, /, %
7	not, ~, унарный -
8	**, ++, --
5.2. Каскадная компиляция модулей
При обнаружении use "module" as M компилятор:

Ищет module.osm в @INC

Если найден — компилирует в module.ocm

Рекурсивно обрабатывает зависимости модуля

Загружает константы модуля

5.3. Парсинг чисел
Обычный режим: все числа → V_NUMBER

Режим -e:

3.14f → V_FLOAT

1.5d3 → V_EDECIMAL

42 или 3.14 → V_NUMBER

5.4. Источниковые комментарии (режим -s)
При компиляции с флагом -s после каждой значимой инструкции в байт-код добавляется # с исходной строкой. VM пропускает строки, начинающиеся с #.

6. Операции с памятью
ByteArray: malloc при создании, free при deallocate() или завершении VM

Хеш: рекурсивное освобождение всей цепочки сегментов

Строки: хранятся в пуле строк, живут до завершения VM

V_NUMBER: хранится по значению (16 байт в Value), без аллокаций

V_FLOAT: хранится по значению (double в Value.data)

V_EDECIMAL: динамическая структура, требует deallocate()

7. Ограничения
Максимальный размер стека: 4096 значений

Максимальный размер пула строк: 256 записей

Максимальное количество модулей: 256

Максимальное количество функций: 1024

Максимальное количество открытых файлов: 256

Длина массива фиксирована при создании

Размер байт-кода: до 256 КБ
